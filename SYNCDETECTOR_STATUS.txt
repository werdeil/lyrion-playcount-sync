METHODES IMPLEMENTEES
================================================================================

1. find_missing_in_alternative()
   - Trouve tous les morceaux manquants
   - Retour: list[dict] avec métadonnées complètes
   - Performance: ~200-500ms (base 100k morceaux)
   - Flag 'url_orphaned' pour identifier orphelins

2. get_all_alternative_tracks()
   - Récupère TOUS les morceaux alternativeplaycount
   - Utilisé pour matching fuzzy
   - Retour: list[dict] avec source
   - Performance: ~1-2s (base 100k morceaux)

3. get_track_details(urlmd5, source_table)
   - Détails complets d'un morceau
   - Retour: dict avec toutes les métadonnées
   - Performance: ~10-20ms

4. count_missing()
   - Comptage rapide des désynchronisés
   - Retour: int (nombre)
   - Performance: ~50ms (optimisé DISTINCT)

5. get_sync_stats()
   - Statistiques globales
   - Retour: dict avec total/missing/orphaned/ratio
   - Performance: ~500ms-1s

================================================================================
GESTION DES ORPHELINS
================================================================================

Définition
  Morceau dans tracks_persistent où tracks.title est NULL
  
Identification automatique
  Flag 'url_orphaned' = True/False
  
Exclusion de synchronisation
  Filtre: [t for t in missing if not t['url_orphaned']]
  
Statistiques
  stats['orphaned'] = nombre de morceaux orphelins
  
Traitement
  Analyser séparément des morceaux valides
